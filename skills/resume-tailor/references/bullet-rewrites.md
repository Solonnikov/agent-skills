# Bullet rewrites

Most résumé weakness is at the bullet level. Fix bullets, the résumé fixes itself.

## The STAR pattern (compact form)

Situation / Task / Action / Result — compressed into one bullet:

```
[Action verb] [what you did] — [for whom/why] — [measurable result].
```

Examples:

```
❌ Responsible for the authentication system.
✅ Rebuilt the authentication system, cutting login latency 60% (1.2s → 480ms) for 3M monthly users.

❌ Worked on the onboarding flow.
✅ Redesigned the onboarding flow; new-user activation rose 18% (from 34% to 52% within 7 days).

❌ Led a team of 5 engineers.
✅ Led a 5-person team through a 6-month platform migration, delivered on schedule with zero customer-visible downtime.
```

The pattern: verb first, specific action, specific result. Keep it to one line if possible, two lines max.

## Strong verb list

Use these. Avoid "worked on", "helped with", "responsible for", "participated in".

### Led / owned

- Led, directed, orchestrated, spearheaded, championed, drove, owned, managed.

### Built

- Built, designed, architected, engineered, developed, created, shipped, delivered, launched, deployed.

### Improved

- Improved, optimized, refactored, streamlined, modernized, accelerated, reduced, increased, scaled, migrated.

### Saved

- Reduced, cut, saved, eliminated, prevented, consolidated.

### Influenced

- Influenced, persuaded, aligned, convinced, negotiated, facilitated, mentored, coached.

### Analyzed

- Analyzed, evaluated, assessed, diagnosed, identified, surfaced, quantified, measured.

### Taught

- Mentored, trained, onboarded, coached, presented, documented, published.

Don't chain verbs ("Led, managed, and owned..."). One strong verb. Let the action do the work.

## Quantify — the three-level test

Try each level. Use the strongest available.

### Level 1: Absolute numbers

- "Shipped 3 features"
- "Trained 12 engineers"
- "Reviewed 400+ PRs"

### Level 2: Percentages / ratios

- "Cut latency 40%"
- "Increased conversion 2.3x"
- "Reduced error rate from 3.2% to 0.4%"

### Level 3: Business impact

- "Cut AWS spend $380k/year"
- "Prevented an outage that would have affected 1M customers"
- "Unblocked a feature that contributed 8% to Q3 revenue"

When you have Level 3, use it. Otherwise Level 2. Only fall back to Level 1 if the percentage / impact isn't available.

### When numbers aren't available

- "Across 4 teams" (scale)
- "In 2 weeks ahead of schedule" (speed)
- "Zero customer-visible downtime" (risk)
- "Adopted by product, design, and engineering" (reach)

Something measurable is almost always available. Hunt for it.

## Common bullet failures

### "Responsible for" — dead verb

```
❌ Responsible for the payments integration.
```

What did you actually do with the payments integration? Did you build it, improve it, replace it, fix it? Use that verb.

### "Helped with" — credit-dodging

```
❌ Helped with the launch of the new dashboard.
```

What did you help with? Were you the lead, a contributor, a reviewer? Be specific.

```
✅ Shipped the frontend for the new dashboard (3 major components, adopted by all product lines).
```

### "Worked closely with" — nothing said

```
❌ Worked closely with the design team.
```

Working closely with someone is a process, not an accomplishment. What came out of that?

```
✅ Partnered with design on 4 product launches; shipped consistent experiences across 3 platforms.
```

### No specific scope

```
❌ Built a recommendation engine.
```

A recommendation engine can be a 200-line heuristic or a multi-year team effort. Specify:

```
✅ Built and deployed a collaborative-filtering recommender in 8 weeks; powers the "Related items" module across 2M daily sessions.
```

### Too much technical detail

```
❌ Implemented a Redis-backed cache with LRU eviction, TTL of 300 seconds, read-through and write-behind semantics, sharded by user_id hash.
```

Recruiters glaze over. Impact and scale matter more than implementation flavor:

```
✅ Implemented a caching layer that cut database load 70% at peak and reduced p95 page load from 1.8s to 400ms.
```

The technical detail lives in the interview, not the bullet.

### Multiple ideas in one bullet

```
❌ Refactored the monolith, onboarded two interns, led weekly standups, and presented at a conference.
```

That's four bullets. Make them four bullets, each with their own action + result. Or cut the weaker ones.

### Leading with "I" or your name

```
❌ I shipped the feature.
```

Drop the "I". First person is implied.

```
✅ Shipped the feature.
```

## Length

- Each bullet: **one or two lines**, max.
- Bullets per role: **3–6**, usually. More for recent / senior roles; fewer for older / less relevant.
- A bullet that stretches to 3+ lines is trying to do too much. Split or cut.

## "But nothing I did is impressive"

Everyone thinks this. Hunt harder:

- Things that took longer than expected → you diagnosed something.
- Things that stopped happening after you joined → you prevented something.
- Things that sped up → you optimized something.
- Knowledge that didn't exist before and does now → you taught or documented.
- Decisions that got made → you influenced.

Also look at:
- PR descriptions you wrote.
- Performance review documents.
- Design docs you authored.
- Incidents you resolved (you diagnosed and fixed; that's valuable).
- Projects shipped on time (you planned and delivered; that's valuable).
- Tech debt you removed (you cleaned up; that's valuable).

Somewhere in your last 2 years of work, there's a specific, quantifiable impact. Find it.
