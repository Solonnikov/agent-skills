# Cuts cheatsheet

Words and phrases that almost always reduce clarity when they appear. Search, consider, usually cut.

## Hedges (weaken claims)

- "I think" / "I feel" / "I believe" — say the thing.
- "maybe" / "perhaps" / "possibly" — cut unless genuinely uncertain.
- "kind of" / "sort of" — almost always delete.
- "a little" / "somewhat" / "slightly" — vague; either commit or drop.

```
Before: "I think maybe we could kind of refactor this."
After:  "Let's refactor this."
```

## Intensifiers (weaken the word they modify)

- "very"
- "really"
- "extremely"
- "highly"
- "absolutely"
- "completely"
- "totally"
- "literally" (unless used literally)

```
Before: "It was really very important to us."
After:  "It was critical to us."
```

## Throat-clearing (delete entirely)

- "I wanted to reach out because..."
- "I hope this email finds you well."
- "As you may know..."
- "It goes without saying..."
- "Needless to say..."
- "At the end of the day..."
- "In my opinion..." / "From my perspective..."

These stall the reader. Just start.

## Filler phrases (paraphrase to one word)

| Before | After |
|--------|-------|
| "in order to" | "to" |
| "due to the fact that" | "because" |
| "at this point in time" | "now" |
| "in the event that" | "if" |
| "for the purpose of" | "for" |
| "prior to" | "before" |
| "subsequent to" | "after" |
| "in the majority of cases" | "usually" |
| "has the ability to" | "can" |
| "make a decision" | "decide" |
| "provide an explanation" | "explain" |
| "come to the conclusion" | "conclude" |
| "give consideration to" | "consider" |

## Vague qualifiers

- "various"
- "a number of"
- "several"
- "some"
- "a variety of"

Replace with a count or a concrete list.

```
Before: "We have various use cases."
After:  "We have 3 use cases: auth, payments, and search."
```

## Redundant pairs

- "each and every" → "every" or "each"
- "first and foremost" → "first"
- "null and void" → "void"
- "true and accurate" → "accurate"
- "safe and secure" → "secure" (or "safe")
- "refer back to" → "refer to"
- "return back" → "return"
- "new innovation" → "innovation"

## "The ... of" constructions

```
Before: "the implementation of the feature"
After:  "implementing the feature" or "the implementation"

Before: "the decision of the team"
After:  "the team's decision"
```

## Zombie nouns

Verbs turned into nouns, usually ending in -tion, -sion, -ment, -ance:

```
Before: "We did an evaluation of the options."
After:  "We evaluated the options."

Before: "The application of this rule..."
After:  "Applying this rule..."
```

## "That" that doesn't earn its keep

```
Before: "He said that he would call."
After:  "He said he would call."

Before: "The fact that we shipped..."
After:  "That we shipped..." or "We shipped, and..."
```

Leave "that" when removing it creates ambiguity. Cut it when it's noise.

## AI tells to look for

Phrases LLMs overuse that real humans rarely write:

- "It's worth noting that..."
- "In today's [anything] landscape..."
- "This is a game-changer."
- "delve into"
- "tapestry" (as a metaphor)
- "leveraging the power of..."
- "unlock the potential of..."
- "seamlessly integrate"

Cut these even if they're technically accurate. They signal generic.

## Weasel words

Specific-sounding claims that commit to nothing:

- "Studies show..."
- "It is widely believed..."
- "Experts agree..."
- "Many users report..."

Either cite the study or drop the claim.
