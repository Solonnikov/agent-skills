# Testing

Anchor's default test runner is Mocha/TypeScript against a local validator via `anchor test`. It's the full integration path — real validator, real SOL, real time. Fast enough for small projects; painful for large ones, where LiteSVM or `solana-bankrun` are worth the migration.

## Anchor default: `anchor test`

```bash
anchor test                  # starts local validator, deploys, runs TS tests
anchor test --skip-deploy    # re-use previous build
anchor test --skip-local-validator    # use an already-running validator
```

Tests live in `tests/*.ts` and use Mocha.

## Smoke test pattern

```ts
// tests/my-program.ts
import * as anchor from '@coral-xyz/anchor';
import { Program } from '@coral-xyz/anchor';
import { PublicKey, Keypair, LAMPORTS_PER_SOL } from '@solana/web3.js';
import { MyProgram } from '../target/types/my_program';
import { expect } from 'chai';

describe('my-program', () => {
  anchor.setProvider(anchor.AnchorProvider.env());
  const program = anchor.workspace.MyProgram as Program<MyProgram>;
  const provider = anchor.AnchorProvider.env();

  const [factoryPda] = PublicKey.findProgramAddressSync(
    [Buffer.from('factory')],
    program.programId,
  );

  before(async () => {
    // Ensure payer has SOL on localnet.
    const sig = await provider.connection.requestAirdrop(
      provider.wallet.publicKey,
      10 * LAMPORTS_PER_SOL,
    );
    await provider.connection.confirmTransaction(sig);
  });

  it('initializes the factory', async () => {
    await program.methods
      .initFactory({ feeBps: 250 })
      .accounts({
        factory: factoryPda,
        payer: provider.wallet.publicKey,
        systemProgram: anchor.web3.SystemProgram.programId,
      })
      .rpc();

    const factory = await program.account.factoryState.fetch(factoryPda);
    expect(factory.feeBps).to.equal(250);
    expect(factory.owner.toBase58()).to.equal(provider.wallet.publicKey.toBase58());
  });

  it('rejects invalid fee', async () => {
    // Second init should fail — account already exists.
    try {
      await program.methods
        .initFactory({ feeBps: 99_999 })
        .accounts({ /* ... */ })
        .rpc();
      expect.fail('should have thrown');
    } catch (err: any) {
      expect(err.toString()).to.include('already in use');
    }
  });
});
```

Key patterns:
- `anchor.workspace.MyProgram` loads the IDL automatically from `target/idl/`.
- `Program<MyProgram>` type comes from `target/types/my_program.ts`, generated by `anchor build`.
- PDAs derived via `PublicKey.findProgramAddressSync` with the same seeds you used in Rust.
- `.accounts({...})` mirrors the `#[derive(Accounts)]` struct.

## Asserting custom errors

Anchor errors come back as structured objects:

```ts
import { AnchorError } from '@coral-xyz/anchor';

it('rejects non-owner calls', async () => {
  const nonOwner = Keypair.generate();
  try {
    await program.methods
      .updateCasino(/* ... */)
      .accounts({ owner: nonOwner.publicKey, /* ... */ })
      .signers([nonOwner])
      .rpc();
    expect.fail('should have thrown');
  } catch (err: any) {
    const anchorErr = AnchorError.parse(err.logs);
    expect(anchorErr?.error.errorCode.code).to.equal('NotCasinoOwner');
  }
});
```

The `errorCode.code` matches the variant name in your `#[error_code]` enum.

## Handling SPL tokens in tests

```ts
import { createMint, createAccount, mintTo, getAccount } from '@solana/spl-token';

const mint = await createMint(
  provider.connection,
  provider.wallet.payer,      // fee payer
  provider.wallet.publicKey,  // mint authority
  null,                       // freeze authority
  6,                          // decimals
);

const userTokenAccount = await createAccount(
  provider.connection,
  provider.wallet.payer,
  mint,
  user.publicKey,
);

await mintTo(
  provider.connection,
  provider.wallet.payer,
  mint,
  userTokenAccount,
  provider.wallet.publicKey,
  1_000_000n,  // 1 token at 6 decimals
);
```

## LiteSVM / Bankrun — the faster alternative

Running `solana-test-validator` per test suite is slow. For larger test suites, switch to [LiteSVM](https://github.com/LiteSVM/litesvm) or `solana-bankrun`:

- In-memory, no validator process.
- 10–100x faster test cycles.
- Supports time-travel (`warp_to_slot`) which `solana-test-validator` doesn't.

Migration is mostly mechanical — replace the `AnchorProvider.env()` setup with the LiteSVM client. Worth doing once a test suite passes ~20 cases.

## What to test

- **Every instruction, happy path.** Does it do what it says?
- **Every constraint, reject path.** Feed it a bad account; confirm it returns the specific error.
- **Every access-control branch.** Owner vs non-owner, admin vs non-admin.
- **State after mutation.** Fetch the account; assert every field.
- **Events.** If you emit Anchor events, parse them from transaction logs.
- **Edge cases.** Zero amounts, max values, accounts that don't exist yet, accounts at rent-exempt minimum.

## What not to test

- Anchor's own macros — they're tested upstream.
- System program transfers at primitive level — focus on your business logic.
- Every possible combination of SOL/SPL — one integration test per variant is enough if the unit of logic is shared.
